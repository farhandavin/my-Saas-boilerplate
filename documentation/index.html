Documentation

Welcome to the Ultimate SaaS Boilerplate documentation. This guide will help you set up, configure, and launch your SaaS application in minutes.

Note: This boilerplate uses the PERN stack (PostgreSQL, Express, React, Node.js).

1. Prerequisites

Before you begin, ensure you have the following installed:

  • Node.js (v18 or higher)
  • PostgreSQL (Running locally or a cloud instance like Supabase/Neon)
  • Stripe Account (For payments)
  • Google Cloud Project (For OAuth & Gemini AI)

2. Installation

We've included an automated script to make setup effortless.

Option A: Automated Setup (Recommended)

node setup-project.js

This command installs dependencies for Root, Backend, and Frontend automatically.

Option B: Manual Installation

# 1. Install Root Dependencies
npm install

# 2. Install Backend
cd backend && npm install
cp .env.example .env

# 3. Install Frontend
cd ../frontend && npm install

3. Detailed API Key Guide

Before configuring your environment file, you need to obtain the necessary credentials. Follow these steps carefully.

A. Authentication Setup

1. Generate JWT_SECRET

This key verifies user sessions. Generate a secure string via your terminal:

openssl rand -base64 32

Copy the output and save it for the next step.

2. Google OAuth (Sign in with Google)

  1. Go to the Google Cloud Console.
  2. Create a New Project (e.g., "SaaS Boilerplate").
  3. Navigate to APIs & Services > OAuth consent screen. Select External and fill in the required app details.
  4. Navigate to Credentials > Create Credentials > OAuth client ID.
  5. Select Web application.
  6. Under Authorized redirect URIs, add: 
    http://localhost:5001/api/auth/google/callback
  7. Copy the Client ID and Client Secret.

B. Stripe Payments Setup

Important: Ensure you are in "Test Mode" (Orange toggle in Stripe Dashboard).

1. API Keys

  1. Log in to Stripe Dashboard.
  2. Go to Developers > API keys.
  3. Copy the Publishable key (starts with pk_test_).
  4. Reveal and copy the Secret key (starts with sk_test_).

2. Webhook Secret

To test subscription updates locally, use the Stripe CLI:

# 1. Login
stripe login

# 2. Start Listener
stripe listen --forward-to localhost:5001/api/webhook

Copy the signing secret (starts with whsec_) from the terminal output.

C. AI & Email Services

1. Google Gemini AI

  1. Visit Google AI Studio.
  2. Click Get API key > Create API key.
  3. Select your Google Cloud project and copy the key.

2. Resend Email (Optional)

  1. Sign up at Resend.com.
  2. Create an API Key with "Sending Access" and copy it.

4. Environment Configuration

Now that you have your keys, open the .env file in the backend/ directory and paste the values:

# Authentication
JWT_SECRET=your_openssl_string
GOOGLE_CLIENT_ID=your_google_id
GOOGLE_CLIENT_SECRET=your_google_secret

# Stripe
STRIPE_PUBLISHABLE_KEY=pk_test_...
STRIPE_SECRET_KEY=sk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...

# AI
GEMINI_API_KEY=your_gemini_key

# Database
DATABASE_URL="postgresql://user:password@localhost:5432/mydb"

5. Database Setup

Once your .env is ready with the DATABASE_URL, initialize the schema.

cd backend
npx prisma generate
npx prisma db push

6. Stripe Product Sync

To finalize subscriptions, you must sync your Stripe Product IDs with the code:

  1. Create two products in Stripe Dashboard: Pro and Team.
  2. Copy the Price ID (e.g., price_1Xy...) for each.
  3. Update backend/src/controllers/paymentController.js with these IDs.
  4. Update frontend/src/pages/Dashboard.jsx with these IDs.

7. Running the App

Start both frontend and backend with a single command from the root:

npm run dev
  • Frontend: http://localhost:5173
  • Backend: http://localhost:5001

8. Feature Usage

  • AI Generator: Go to "AI Tools" in Dashboard. Prompts are processed via Google Gemini.
  • Team Management: Create organizations and manage members via the "Team" tab.
  • Billing: Test upgrades using Stripe Test Cards (e.g., 4242 4242...).

9. Deployment

The project includes vercel.json for easy deployment.

  • Frontend: Deploy to Vercel/Netlify. Set VITE_API_URL to your backend URL.
  • Backend: Deploy to Render, Railway, or Vercel. Ensure all environment variables are set in the cloud dashboard.
